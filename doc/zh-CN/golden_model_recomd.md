本文讨论 AI 加速器验证中 Golden Model 的推荐实践。

就如同在 tilling_schedule.md 中说的，软件分为 逻辑和调度，而高层软件语言描述比如 pytorch 中，只有逻辑部分。所以我们的策略是

- 首先肯定要复用 Pytorch 生态，Golden Model 写法兼容 Pytorch
- 在 Pytorch 基础上补足调度的信息
- Pytorch 算法支持拼起来跑端到端


比如一个加法，这是算法原型

```python
def add(a, b):
    return a + b
```

这包含了什么信息
- 包含了数据 a 和 b 的具体数值和逻辑操作

这缺少了什么信息
- 缺少了硬件调度的 tilling 策略，单位时间步计算数据量是多少，切分成多少块？
- 如何涉及 buffer 交互数据，还有地址信息
- 以及单元的指令信息，包含数据维度操作类型之类


因此我们要实现两种算法模型，一种是纯算法层面的 add ，另一种则包括硬件补助的信息用以进行比较
